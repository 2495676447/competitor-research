3. **300+ 专利 + 亚洲品牌 500 强** — 数字信任

| CTA | 目标 |
|-----|------|

## 5. FAQ / 常见问题分析

### FAQ 页面结构

亿合没有独立的 FAQ 页面，而是在"新闻资讯"栏目下设"常见问题"子栏目，内容以**科普文章**形式呈现而非 Q&A 列表。

已发现的 FAQ 类内容：

| 标题 | 类型 |
|------|------|
| 打造"五感"体验，亿合门窗全新定义美好生活家 | 品牌软文（非真 FAQ） |
| 家居门窗安全常见的问题 | 安全科普 |
| 铝合金门窗厂家的网络品牌建设如何才能做好呢 | B2B 营销科普 |
| 门窗企业建网站需重视视觉体验建设 | B2B 营销科普 |

### FAQ 写法特点

| 维度 | 分析 |
|------|------|
| **格式** | 文章式而非 Q&A 列表式 — SEO 友好但用户查找效率低 |
| **语调** | 品牌推广语气偏重，解答性质偏弱 |
| **数据密度** | 低 — 缺少具体参数（如质保年限、标准编号） |
| **CTA 策略** | 几乎无 — 文章末尾无下一步引导 |
| **SEO 优化** | 文章标题有做关键词，但非结构化 FAQ Schema |

### 关键缺失

与 Andersen 的 FAQ 知识库相比，亿合的 FAQ 存在三个严重不足：

1. **非结构化** — 没有 Q&A 格式，用户无法快速找到答案
2. **无 Schema 标记** — 错失 Google FAQ 富文本搜索结果展示机会
3. **内容漂移** — 大量"常见问题"实际是品牌软文，不是真正的客户问题解答

### 可复用技巧

- 虽然 FAQ 弱，但亿合的**产品科普文章**可以作为 FAQ 的补充材料思路
- "家居门窗安全常见的问题"这类选题方向值得学习 — 从用户痛点出发组织内容

## 6. 转化路径分析

### 双轨商业模式

```
B2B 招商轨（核心）:
  首页招商入口 → 招商政策页 → 400 热线 → 加盟签约 → 门店开业

```

**关键洞察**: 亿合的网站首要目标是**招商**，不是卖窗给业主。这和 Andersen（B2C 业主导向）和中国出口商（B2B 海外采购商导向）有本质区别。理解这个才能理解它的网站设计逻辑。

### 联系/询盘

- 400 热线电话（首页显著）
- 企业邮箱
- 门店查询（引导线下）
- 无在线购买 / 无在线报价

## 8. 对我们的启示

### 🔥 可复用策略

| # | 亿合的做法 | 我们可以怎么做 |
|---|----------|-------------|
| 1 | **"三防三隔"独创标准** — 行业首创一个概念，成为品类代名词 | 创造一个独有技术概念（如"X 层防护体系"），注册为品牌资产 |
| 2 | **高校学术背书** — 清华 + 哈工大 + 华南理工三校合作 | 找一家国际认可的检测机构/大学做联合认证 |

- 欧洲地名命名在海外市场可能适得其反 — 西方买家会觉得"不真实"

- Title 关键词堆砌是 2019 年的 SEO 做法，现在已过时 — 用语义 SEO 替代

## 9. 对我们的具体行动建议

1. **立刻做** — 创造一个独有技术概念（类似"三防三隔"），成为品牌核心差异化标签
2. **立刻做** — 选定一个"英雄产品"集中火力推广，用具体数字证明领先（如市占率、销售额占比）

5. **长期做** — 考虑产品系列命名策略 — 欧洲地名在国内有效，海外需另起命名体系

## B. On-page SEO

### Title 标签

```
首页:    亿合门窗_断桥铝门窗厂家_别墅高端智能门窗定制_全防护门窗
关于:    亿合门窗品牌简介-实力及资质,亿合门窗怎么样？
文章:    [文章标题]_亿合门窗
产品:    [产品名]_亿合门窗
```

**问题**: 首页 Title 堆砌 5 个关键词，2019 年的 SEO 做法，现在百度/Google 都惩罚这种做法。

### Meta Keywords

```
首页 Keywords: 别墅门窗, 断桥铝门窗, 智能门窗, 全防护门窗, 高端门窗, 亿合门窗, EEHE
```

**问题**: Meta Keywords 标签早已被搜索引擎弃用，保留它说明 SEO 维护停滞在 2019 年。

### H1/H2 结构

```
首页:
  H1: [品牌图 — 无文本 H1！❌]
  （首页缺少文本 H1 标签，是非常严重的 SEO 缺陷）

文章页:
  H1: [文章标题]
  H2: 使用较少，结构松散
```

### URL 结构

```
品牌动态:  /article/ppdt_815.html     ← 含数字 ID
常见问题:  /article/cjwt_1091.html    ← 含数字 ID
产品页:    /product/xxx               ← 推测
```

**问题**: URL 使用数字 ID 而非语义化路径（对比 Andersen 的 `/blog/projects/project-name`），SEO 不友好。

## C. 内容资产审计

### 最有价值的页面类型

| 优先级 | 页面类型 | 为何值钱 | 状态 |
|--------|---------|---------|------|
| 🥇 | 常见问题（31 页） | 长尾信息词，量大 | ⚠️ 文章式非 FAQ 式，缺 Schema |
| 🥈 | 品牌动态 | 品牌词 + 事件词 | ✅ 更新频繁 |
| 🥉 | 产品页 | 品类关键词 | ⚠️ 文字少，图多 |
| 4 | 招商加盟页 | B2B 招商转化 | ✅ 结构完整 |
| 5 | 案例展示 | 间接转化 | ⚠️ 图片为主，文字少 |

### SEO 最大问题

1. **首页无 H1 标签** — 这是 On-page SEO 最基础的错误
2. **URL 无语义化** — 数字 ID 路径无关键词价值
3. **无 FAQ Schema** — 31 页常见问题文章全部错失富文本展示
4. **Title 关键词堆砌** — 2019 年做法，已过时
5. **移动端 SEO 未验证** — 需要测试

## D. FAQ SEO 策略

### 当前状态

有 31 页"常见问题"文章，这是亿合**最大的 SEO 资产**。但：
- ❌ 格式是文章式，不是 Q&A 列表式
- ❌ 无 `FAQPage` Schema
- ❌ 品牌软文混在其中，稀释了 FAQ 信号
- ✅ 内容量大（31 页），选题方向正确

### 应该怎么做（改进建议）

```
当前格式:  文章标题 + 品牌软文式正文
改进为:    H2 = 分类 / H3 = 具体问题 Q / P = 简短答案 A
加上:      FAQPage JSON-LD Schema
```
